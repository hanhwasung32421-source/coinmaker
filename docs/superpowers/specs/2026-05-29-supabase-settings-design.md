# Supabase 설정 저장/복원 + 배경 컨트롤 개선 (Design)

## 목표
1. 배경 줌을 0.1% 단위(step=0.001)로 조절 가능하게 한다.
2. 배경 네비게이터 이동 시 X/Y 좌표를 표시하고, X/Y 입력으로 원하는 좌표로 이동 가능하게 한다.
3. 모든 설정값을 Supabase에 저장하고, 다음에 페이지를 열면 자동으로 불러와 적용한다.

## 범위(저장 대상)
- 숫자 입력: 수익률 최소/최대, 최소/최대 수익금(만원), 코인 텍스트, 레버리지, 진입가격, 생성 개수, 파일명 접두사
- 포지션: LONG/SHORT
- 배경: bgZoom, bgShiftX, bgShiftY
- 텍스트 미세조정: textAdjust(각 텍스트 id별 dx/dy/size/bold)

## UI 변경
### 1) 배경 줌
- 기존 `inpBgZoom`의 step을 `0.001`로 변경한다. (0.1% 단위)
- 표시 값은 3자리 소수(예: 1.000) 유지.

### 2) 배경 이동(X/Y)
- 기존 버튼(위/아래/왼쪽/오른쪽/원점)은 유지.
- 추가:
  - 현재 좌표 표시: `X: {bgShiftX}, Y: {bgShiftY}`
  - 입력칸 2개: `bgX`, `bgY`
  - 적용 버튼: 입력값을 bgShiftX/bgShiftY로 반영

## Supabase 저장/복원
### 데이터 모델
- 테이블: `public.app_settings`
- 공통 설정은 단일 행 `id='global'`에 저장한다.
- 컬럼:
  - `id text primary key`
  - `data jsonb`
  - `updated_at timestamptz`

### 클라이언트 동작
- 시작 시: `select`로 `id='global'` 데이터를 가져와 UI/내부 상태에 적용 후 `renderAll()`
- 변경 시: 사용자 입력/버튼 조작이 발생할 때마다 데이터 스냅샷을 만들고, 300~500ms 디바운스로 `upsert(id='global')`

### 보안/정책
- 사용자가 “공개 테이블 허용(anon 읽기/쓰기 가능)”을 선택한 상태로 구현한다.
- 주의: anon 키가 포함된 프론트엔드는 누구나 쓰기 가능하므로 덮어쓰기 위험이 있다.

## 오류 처리 / 복원 전략
- Supabase 연결 실패 시:
  - 기존 로컬 기본값(localStorage/기본값)을 유지하고 앱은 계속 동작.
  - 저장 실패는 조용히 무시(콘솔 로그 정도).
- 데이터 스키마가 일부 누락되어도, 누락 값은 현재 기본값을 유지한다(부분 merge).

## 검증(수동)
1. 줌 step이 0.001로 동작하고 렌더링이 정상인지 확인
2. 배경 이동 버튼 클릭 시 X/Y 표시가 즉시 갱신되는지 확인
3. X/Y 입력 후 적용 시 정확히 이동하는지 확인
4. 값 변경 → 새로고침 → 동일 값이 자동 적용되는지 확인
